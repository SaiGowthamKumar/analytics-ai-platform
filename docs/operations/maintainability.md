# Maintainability

**Purpose:** Preserve long-term change safety.

**Owner:** Engineering Lead

Maintain small cohesive modules, stable public contracts, high-signal tests, structured observability, dependency hygiene, documented ownership, and ADR history. Prefer deletion over dead abstractions. Deprecate contracts with migration windows and telemetry. Keep documentation synchronized with behavior; docs changes are part of the definition of done for material changes.

## Future expansion

Add code-health metrics, ownership matrix, and deprecation policy.
