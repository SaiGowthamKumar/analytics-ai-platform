# ADR-004: FastAPI

**Purpose:** Record the proposed backend delivery adapter framework.

**Owner:** Principal Architect

**Status:** Proposed

## Context

The backend needs typed HTTP APIs and an async-capable Python ecosystem.

## Decision

Use FastAPI behind application ports, subject to approval.

## Alternatives

Django, Flask, Node.js services.

## Consequences

Fast development and OpenAPI support; framework stays at adapter edge.

## Tradeoffs

Python runtime conventions versus other ecosystem choices.

## Future expansion

Record API platform constraints and final acceptance decision.
